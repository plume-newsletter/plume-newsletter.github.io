title: มากกว่าฟีเจอร์พื้นฐาน
description: พาชมฟีเจอร์ที่มาพร้อม Plume เวอร์ชันปัจจุบัน ตั้งแต่ผู้ช่วย AI ไปจนถึงทีมงานและ block editor
---

ฟีเจอร์ทั้งหมดนี้มาพร้อมกับเวอร์ชันปัจจุบันแล้ว คู่มือเจาะลึกรายฟีเจอร์กำลังจะตามมา — หน้านี้เป็นเพียงการพาชมสั้น ๆ ว่ามีอะไรให้ใช้บ้าง และแต่ละอย่างอยู่ตรงไหนใน API

## ผู้ช่วย AI

ผู้ช่วยที่ขับเคลื่อนด้วย Claude ถูกสอดแทรกอยู่ทั่วทั้งแอป: ช่วยเขียนเนื้อหาใหม่ พูดคุยเกี่ยวกับ campaign แนะนำหัวข้ออีเมล เจาะลึก insight ของ campaign และแปลงภาษาธรรมชาติให้เป็นเงื่อนไข segment ตั้งค่า Anthropic API key ได้ที่ **Settings** ซึ่งจะถูกจัดเก็บแบบเข้ารหัส Endpoint อยู่ใต้ `/api/ai/*` เช่นเดียวกับฟีเจอร์ AI อื่น ๆ ของ Plume ฟีเจอร์นี้เป็นทางเลือกล้วน ๆ — จะไม่มีอะไรทำงาน และจะไม่มี key ใดหลุดออกจาก config ของ workspace คุณเลย จนกว่าคุณจะตั้งค่ามันเอง

### Campaign copilot (ผู้ช่วยร่าง campaign)

แทนที่จะเริ่มจาก campaign เปล่า ๆ ให้อธิบายสิ่งที่ต้องการในไดอะล็อก "New campaign" — "✨ Or describe it and let AI draft everything" — แล้ว editor จะเปิดตรงเข้าสู่แผง copilot พร้อม draft ฉบับเต็มที่ถูกใส่ไว้ให้แล้ว: หัวข้อและบล็อกเนื้อหาถูกเขียนไว้ พร้อมให้ตรวจทาน

จากจุดนั้นเป็นการสนทนา: คุยต่อในแผงเพื่อปรับ draft ("ทำให้สั้นลง", "เพิ่มโค้ดส่วนลด", "โทนสบาย ๆ มากขึ้น") แต่ละคำตอบจะแทนที่ draft ใน editor พร้อมปุ่ม **Undo** แบบขั้นตอนเดียวถ้าผลลัพธ์ไม่ตรงใจ เมื่อคำขอของคุณสื่อถึงกลุ่มเป้าหมาย — "ส่งให้คนที่ไม่ได้เปิดอ่านใน 30 วันที่ผ่านมา" — แผงจะเสนอกลุ่มเป้าหมายเป็นสรุปภาษาธรรมชาติพร้อมจำนวน subscriber และมีปุ่ม **Save audience** ที่แปลงเป็น segment ให้เลือกใช้ตอนส่งได้ นอกจากนี้ยังเสนอ **คำแนะนำช่วงเวลาส่ง** ที่อ้างอิงจาก analytics ของคุณเอง ไม่ใช่ heuristic ทั่วไป

Copilot จะส่งมอบ draft ให้คุณเท่านั้น — ไม่เคยส่ง campaign เอง การส่งยังคงเป็นขั้นตอนที่ต้องตัดสินใจแยกต่างหากจากหน้าต่างส่ง ดู [`POST /ai/campaign`](/th/docs/rest-api/) สำหรับ contract เบื้องหลัง

## Analytics (ภาพรวมข้อมูล)

ภาพรวมการส่งอีเมลระดับเจ้าของ workspace: จำนวน subscriber รวม อัตรา engagement ต้นทุนการส่งโดยประมาณ การเติบโตและปริมาณตามช่วงเวลา ช่วงเวลาส่งที่ดีที่สุด และตัวชี้วัดรายแคมเปญ Endpoint อยู่ใต้ `/api/analytics/*`

## Automations (ระบบทำงานอัตโนมัติ)

การเดินทาง (journey) แบบเชิงเส้นที่ตั้งเงื่อนไขไว้ล่วงหน้า ประกอบด้วยลำดับขั้นตอนส่งและรอที่เรียงกันไว้ Subscriber จะลงทะเบียนเมื่อเกิด `subscriber.confirmed` และถูกเลื่อนผ่านแต่ละขั้นตอนโดย worker เบื้องหลัง Endpoint อยู่ใต้ `/api/automations*`

## การทดสอบ A/B (A/B test)

การทดสอบหัวข้ออีเมล: แบ่ง list ออกเป็นสองส่วน ส่งหัวข้อที่แข่งกันสองแบบไปยังส่วนที่แบ่งไว้ แล้วส่งหัวข้อที่ชนะไปยังส่วนที่เหลือ (holdout) Endpoint อยู่ใต้ `/api/ab-tests*`

## Segment คืออะไร

ตัวกรองที่บันทึกไว้ ซึ่ง compile จากเงื่อนไข JSON ลงเป็น SQL มี live preview แสดงจำนวนที่ตรงเงื่อนไขและตัวอย่างขณะที่คุณสร้าง Endpoint อยู่ใต้ `/api/segments*`

## Signup form (ฟอร์มสมัครสมาชิก)

ฟอร์มสมัครสมาชิกแบบโฮสต์เอง พร้อมหน้า landing page สาธารณะที่ `/f/{id}` จัดการได้ใต้ `/api/signup-forms*`

## Template (เทมเพลตอีเมล)

เลย์เอาต์อีเมลที่ใช้ซ้ำได้ — ทั้ง template เริ่มต้นและดีไซน์ที่คุณบันทึกไว้เอง กดปุ่ม "Use" เพื่อสร้าง draft campaign ขึ้นจาก template นั้นทันที Endpoint อยู่ใต้ `/api/templates*`

## ทีมและ workspace

Workspace แบบผู้ใช้หลายคน พร้อมบทบาท คำเชิญ และการจัดการสมาชิก Endpoint ครอบคลุม `/api/team*`, `/api/workspace` และ `/api/invites/*`

## API key (คีย์สำหรับเรียก API)

Key ที่ผูกกับ workspace สำหรับ REST API สร้างและจัดการได้ใต้ `/api/api-keys*` แล้วนำไปใช้กับ [REST API](/th/docs/rest-api/)

## Block editor (ตัวแก้ไขแบบบล็อก)

Campaign ถูกเขียนขึ้นเป็นอาร์เรย์ของบล็อก ซึ่ง Plume แปลงเป็น HTML ที่ปลอดภัยสำหรับอีเมล พร้อมเวอร์ชัน plain-text คู่กัน render อาร์เรย์บล็อกได้ด้วย `POST /api/blocks/render`

## ภาษาของอินเทอร์เฟซ

แดชบอร์ดรองรับทั้งภาษาอังกฤษและภาษาไทย สลับได้ทุกเมื่อที่ **Settings → General** — การเปลี่ยนแปลงมีผลทันทีและถูกจดจำไว้ต่อเบราว์เซอร์ (ค่าเริ่มต้นสำหรับ session ใหม่คือภาษาอังกฤษ) ปุ่มสลับแบบเดียวกันนี้ยังมีอยู่ที่หน้า login ก่อนที่คุณจะเข้าสู่ระบบด้วย

อยากใช้ Plume ในภาษาของคุณไหม แปล `web/src/i18n/en.json` เป็นภาษาของคุณแล้วเปิด PR เข้ามาได้เลย — เราจะช่วยต่อภาษาใหม่เข้ากับตัวสลับภาษาให้เอง
