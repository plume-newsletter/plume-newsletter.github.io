เชื่อมต่อ brand เข้ากับฟีดของบล็อกคุณ แล้ว Plume จะร่าง campaign ให้ทุกครั้งที่มีโพสต์ใหม่ ไม่มีการส่งอัตโนมัติ — คุณตรวจทานก่อนเสมอ

## เชื่อมต่อฟีด

1. **Brands** → brand ของคุณ → **RSS**
2. วาง URL ฟีดของคุณแล้วเลือกโหมด
   - **Post as-is** — draft จะมีเนื้อหา HTML ของโพสต์ตามต้นฉบับ แก้ไขได้ใน composer
   - **Let AI write it** — AI ของ Plume (ต้องตั้งค่า [AI settings](/th/docs/additional-features/) ก่อน) จะเขียนจดหมายข่าวจากโพสต์ของคุณ ตามคำสั่งประจำ (standing instruction) ที่คุณตั้งไว้ (เช่น "สรุปสั้น ๆ โทนสบาย ๆ ปิดท้ายด้วยลิงก์อ่านเพิ่มเติม") ถ้า AI ใช้งานไม่ได้ คุณจะได้ draft แบบ as-is แทน — โพสต์จะไม่หายไปไหนแน่นอน
3. เชื่อมต่อ Plume จะตรวจสอบฟีดทันที — URL ที่ผิดพลาดจะล้มเหลวให้เห็นตรงนั้นเลย

URL ฟีดที่พบบ่อย

| แพลตฟอร์ม | URL ฟีด |
|---|---|
| WordPress | `https://yourblog.com/feed/` |
| Ghost | `https://yourblog.com/rss/` |
| Hugo / Jekyll / Astro | ส่วนใหญ่เป็น `/index.xml`, `/feed.xml` หรือ `/rss.xml` |

## การ poll ทำงานอย่างไร

- Plume ตรวจสอบฟีดทุก **15 นาที**
- **เฉพาะโพสต์ที่เผยแพร่หลังจากคุณเชื่อมต่อ** เท่านั้นที่จะกลายเป็น draft — การเชื่อมต่อจะไม่มีวันถล่มคุณด้วยโพสต์เก่าในคลัง
- สูงสุด **3 draft ต่อการตรวจสอบหนึ่งครั้ง**: ถ้าฟีดจู่ ๆ รายงานรายการ "ใหม่" จำนวนมาก (ย้ายไซต์ เปลี่ยนปลั๊กอินฟีด) จะสร้าง draft ให้กับ 3 รายการล่าสุดเท่านั้น ส่วนที่เหลือจะถูกข้ามไปถาวร
- ปัญหาการดึงข้อมูลจะแสดงในไดอะล็อก RSS ของ brand และจะหายไปเมื่อการตรวจสอบครั้งถัดไปสำเร็จ

## อยากได้ pipeline แบบกำหนดเอง?

[n8n node](/th/docs/n8n/) ร่วมกับ RSS trigger ในตัวของ n8n ให้คุณควบคุมได้เต็มที่ — กรองโพสต์ จัดหมวดหมู่ต่างกันไปยังต่างลิสต์ หรือเพิ่มขั้นตอนอนุมัติ
