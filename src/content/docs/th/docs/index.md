Plume คือแพลตฟอร์มจดหมายข่าวแบบโอเพนซอร์สที่โฮสต์เอง คุณส่งอีเมลผ่านผู้ให้บริการของคุณเอง — Amazon SES, Mailgun, SendGrid, Postmark, Resend หรือเซิร์ฟเวอร์ SMTP ใดก็ได้ — จ่ายตามต้นทุนของผู้ให้บริการ ไม่ใช่ตามจำนวนสมาชิก และข้อมูลของคุณไม่เคยออกจากเซิร์ฟเวอร์ของคุณ

:::note
เพิ่งเริ่มใช้งาน? ไปที่ [Quickstart](/th/docs/quickstart/) เพื่อติดตั้งให้ใช้งานได้ภายในประมาณห้านาที
:::

## ทำไมต้อง Plume

เครื่องมือจดหมายข่าวแบบ hosted ส่วนใหญ่คิดราคาตามจำนวนสมาชิก ทำให้บิลของคุณพุ่งขึ้นตามขนาดลิสต์ที่โต — ทั้งที่การส่งอีเมลแทบไม่มีต้นทุนเลย Plume แยกสองสิ่งนี้ออกจากกัน: ตัวซอฟต์แวร์ฟรีและเป็นโอเพนซอร์ส ส่วนต้นทุนการส่งเพียงอย่างเดียวคือค่าบริการที่ผู้ให้บริการอีเมลเรียกเก็บ — ต่ำสุดเพียง `$0.10` ต่อ 1,000 อีเมลด้วย Amazon SES

- **เป็นเจ้าของการส่งอีเมลเอง** — บัญชีผู้ให้บริการของคุณเอง ชื่อเสียงของคุณเอง ลิสต์ของคุณเอง — ไม่มีคนกลาง
- **ไบนารีเดียวจบ** — Go binary แบบ static ตัวเดียวที่ฝัง UI ไว้ในตัว บวกกับ Postgres
- **โอเพนซอร์ส** — สัญญาอนุญาต AGPL-3.0 รุ่นโฮสต์เองใช้ฟรีตลอดไป
- **ขยายได้** — ระบบ hook ระดับเฟิร์สคลาส — ไม่ต้อง fork โค้ด

## หลักการทำงาน

Plume จัดการ brand, list และ subscriber ให้คุณเขียน HTML campaign แล้วส่งต่อให้ background worker ทยอยส่งอีเมลไปยัง [ผู้ให้บริการอีเมล](/th/docs/email-providers/) ที่คุณตั้งค่าไว้ ภายใต้อัตราการส่งที่กำหนด ระบบจะเก็บข้อมูลการเปิดอ่านและคลิกจากทุกผู้ให้บริการ และ — ถ้าใช้ Amazon SES — เก็บ bounce และ complaint ด้วย พร้อมระงับที่อยู่อีเมลที่มีปัญหาโดยอัตโนมัติเพื่อปกป้องชื่อเสียงของคุณ

```
connect provider → import → compose → worker → track
```

## สิ่งที่คุณต้องมี

- บัญชีผู้ให้บริการอีเมล — Amazon SES (ประหยัดที่สุดเมื่อส่งปริมาณมาก) หรือผู้ให้บริการ SMTP ใดก็ได้ — ไม่จำเป็นตอนเริ่มต้น เพราะ Plume มี log provider มาให้สำหรับทดสอบในเครื่อง
- ฐานข้อมูล PostgreSQL 14 ขึ้นไป
- Docker หรือเครื่องที่รันไบนารีเดียวได้
- โดเมนที่คุณสามารถเพิ่มระเบียน DNS ได้
