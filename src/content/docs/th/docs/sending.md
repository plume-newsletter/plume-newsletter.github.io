เมื่อคุณส่ง campaign, Plume จะส่งต่อให้ **send worker** เบื้องหลังซึ่งทยอยส่งอีเมล (drip) ไปยังผู้ให้บริการอีเมลที่ตั้งค่าไว้ (SES หรือ SMTP) ภายในอัตราที่กำหนด — ทนทานต่อความล้มเหลวและทำงานต่อได้จากจุดที่ค้าง worker เป็น goroutine ที่รันอยู่ภายในโปรเซส Plume เดียว ไม่ใช่โปรเซสแยกที่ต้อง deploy เอง

## ขั้นตอนการส่ง

1. Campaign ถูกเข้าคิวและ snapshot รายชื่อผู้รับไว้
2. ที่อยู่ที่ถูก suppress จะถูกกรองออก
3. Worker จะรวมข้อความเป็นชุดและเรียกผู้ให้บริการที่ตั้งค่าไว้
4. การส่งถึง การเปิดอ่าน และการคลิก จะสตรีมกลับมาเป็น event

## การเลือกผู้รับ

หน้าต่างส่งมีแท็บ **List** และ **Segment** — เลือกเป้าหมายได้เพียงอย่างเดียว ไม่ใช่ทั้งคู่ การส่งไปยัง list จะส่งถึงทุกคนใน list นั้น (หักลบส่วนที่ถูก suppress ตามด้านล่าง) ส่วนการส่งไปยัง [segment](/th/docs/additional-features/) จะส่งถึงเฉพาะ subscriber ที่ตรงตามเงื่อนไขของ segment ณ ขณะนั้น และเฉพาะผู้ที่มีสถานะ `active` เท่านั้น — segment ที่สร้างจากเงื่อนไขเช่น "เปิดอ่านใน 30 วันที่ผ่านมา" จะไม่มีทางส่งถึงที่อยู่ที่ยัง pending หรือ unsubscribe แล้ว แม้เงื่อนไขของ segment จะตรงกับที่อยู่นั้นก็ตาม การกรอง suppression และการ unsubscribe แบบคลิกเดียวต่อผู้รับทำงานเหมือนกันทุกประการไม่ว่าจะเลือกแบบไหน

## การจำกัดอัตราการส่ง

Worker เคารพอัตราการส่งของผู้ให้บริการของคุณ จะไม่มีการส่งเกินโควตาแบบพรวดเดียว จึงไม่ถูกจำกัด (throttle)

## การ retry

Error ชั่วคราวจากผู้ให้บริการจะถูก retry แบบ backoff ถ้า worker รีสตาร์ทกลางคันระหว่างส่ง มันจะทำงานต่อจากจุดที่ค้างพอดี — ไม่ส่งซ้ำ ไม่ตกหล่น

## การตั้งเวลาส่ง

ตั้งเวลาส่ง campaign ล่วงหน้าได้ หรือให้ [ผู้ช่วย AI](/th/docs/additional-features/) แนะนำช่วงเวลาส่งที่ดีที่สุดจาก engagement ในอดีตของ list คุณ
